=== Fig.2  问题形式化图（三种退化机制对比）===
====================================================================

请绘制Fig.2：PRI Sequence Degradation Mechanisms。

【内容要求】
用一张图展示三种退化过程（Miss / Spurious / Mix），每种退化用
一行展示，共三行，外加一行干净序列（Ground Truth），合计四行。

整体布局（四行，每行从左到右）：

行1 — Ground Truth：
  用一条横向时间轴（t轴），在固定时刻绘制8~10个竖向脉冲，
  脉冲高度相同，等间隔，PRI间隔标注为"T"。
  行标：(a) Ground Truth

行2 — Miss（脉冲丢失）：
  从Ground Truth复制，随机删除2~3个脉冲（在对应位置打×表示
  脉冲消失），删除脉冲后，其两侧的间隔合并变为"2T"，
  用红色双箭头标注这个加倍的间隔。
  行标：(b) Missing Pulses (miss rate ≈ 20%)

行3 — Spurious（虚假脉冲）：
  从Ground Truth复制，在2~3个原有间隔中间插入新的虚假脉冲
  （用橙色△标注），使得那些位置的PRI间隔变小（约原值一半），
  用橙色标注"≈T/2"。
  行标：(c) Spurious Pulses (spurious rate ≈ 10%)

行4 — Mix（混合）：
  同时包含丢失（打×）和虚假脉冲（橙色△），以及脉冲位置轻微
  抖动（表示高斯噪声，脉冲不是精确对齐到grid）。
  行标：(d) Mixed Degradation (Miss + Spurious + Noise)

每行右侧绘制一个小坐标系，用折线图显示对应PRI值序列（x轴为
脉冲序号，y轴为PRI值μs），量化展示退化效果：
  · Ground Truth：平坦水平线
  · Miss：出现峰值（某些值约为原值的2倍）
  · Spurious：出现谷值（某些值约为原值的一半，即很小的PRI）
  · Mix：同时有峰值和谷值

请绘制Fig.10：Loss Function Design of DiffSeqPRI。

【内容要求】
展示三个损失分量的计算路径，采用从左到右的数据流图形式。

整体布局（左→右数据流）：

左侧输入（三个圆角框并列）：

  → 输出：[Predicted Clean Embedding pred_x₀]（分叉为两路）

第一路（上方，蓝色）：
  pred_x₀ → [MSE Loss]（与Ground-Truth Embedding x₀对比）
  公式标注：L_mse = ‖pred_x₀ - x₀‖² （仅target位置）

  pred_x₀ → [LM Head Linear(128→1505)] → Logits
  → [Cross-Entropy Loss（CE）]（与y对比）
  公式标注：L_ce = CrossEntropy(logits, y) （target位置，含label smoothing=0.1）
  在CE框下方补充说明："No time weighting (loss_mode: mse+ce_no_tw)"

第三路（下方，红色）：
  从第二路的Logits分叉 → [END Token Loss]
  只在target段中[END]Token位置计算CE
  公式标注：L_end = CE(logits[END pos], END_ID)

三路汇总到右侧的总损失框：
  [Total Loss L]
  公式：L = L_mse + 0.3 · L_ce + 0.5 · L_end
  权重用彩色系数标注（0.3橙色，0.5红色，1.0蓝色）

右侧添加一个"Mask说明"小图：

请绘制Fig.12：Training and Inference Workflow of DiffSeqPRI。

【内容要求】
左右两列对比，左列为Training Workflow，右列为Inference Workflow，
两列各用竖向步骤流程图表示，步骤框从上到下排列，用带序号的圆角
矩形框表示，框间用箭头连接。

左列 — Training Workflow（8步）：

  ① [Data Preparation]
     · Load (observed PRI, clean PRI) pair
     · Quantize to Token IDs via PRI Tokenizer

  ② [Seq2Seq Layout Construction]
     · Concatenate: [START]SRC[END][SEP]TGT[END][PAD]
     · Build input_mask (1 for target tokens only)

  ③ [Embedding Lookup]
     · Token IDs → Embeddings x₀ ∈ ℝ^{L×128}

  ④ [Random Timestep Sampling]
     · Sample t ~ Uniform{1, ..., T=50}

  ⑤ [Forward Noising (target only)]
     · x_t = √ᾱ_t·x₀ + √(1-ᾱ_t)·ε (masked to target segment)
     · Source segment remains x₀_src (unchanged)

  ⑥ [TransformerEncoder Prediction]
     · Input: x_t, timestep t (+ position embedding)
     · Output: pred_x₀

  ⑦ [Loss Computation & Backprop]
     · L = L_mse + 0.3·L_ce + 0.5·L_end
     · AdamW optimizer, grad clip=1.0

  ⑧ [EMA Update]
     · shadow ← 0.999·shadow + 0.001·θ

右列 — Inference Workflow（7步）：

  ① [Input Preparation]
     · Load observed PRI sequence
     · Quantize to Token IDs

  ② [Seq2Seq Layout (Test Mode)]
     · [START]SRC[END][SEP] + NOISE(length unknown)
     · target_mask: all positions after SEP → 1

  ③ [Initialize Noisy Target]
     · x_T ← [x_src_emb | z] where z ~ N(0,I)

  ④ [DDIM Sampling Loop (5 steps)]
     · For t = T, ..., 0:
       (a) pred_x₀ = TransformerEncoder(x_t, t)
       (b) Rounding: x₀ ← nearest_token(pred_x₀)
       (c) Compute x_{t-1} via DDIM update
       (d) Restore source segment to x_src_emb

  ⑤ [Token Decoding]
     · Nearest-neighbor search: embeddings → Token IDs
     · Find first [END] token after [SEP]

  ⑥ [Variable-Length Output Extraction]
     · Extract tokens between [SEP] and [END]
     · Decode Token IDs → PRI values (μs)

  ⑦ [Reconstructed PRI Sequence]
     · Output: clean PRI sequence of inferred length n

  · 标题加粗，细节用小字
  · 重要步骤（如DDIM循环、END token检测）用颜色强调框（橙色或红色框）
